Elastic SVGs

By default, <svg> blocks (Scalable Vector Graphics) do not resize to the window and typically have hard-coded dimensions. You can easily fix that with elasticSVG, a small, pure-JavaScript library maintained by Chris Wilson and open-sourced by TIME Magazine.

Installation

In Node:

npm install elastic-svg --save import elasticSVG from 'elastic-svg';

In the browswer, clone the repo and include ./dist/elasticSVG.min.js in the document: <script src='./dist/elasticSVG.min.js'></script>

How it works

The traditional approach is to give your <svg> a width of 100% and use the SVG viewBox property to automatically scale the graphic up or down. `elasticSVG` does that automatically with the `resize` parameter. You can see it in action below by resizing the window or by dragging the handle in the lower right corner:

var base = elasticSVG("#my_container_div", {
    resize: "auto"
});

But as you see here, this is not always very effective, particularly which a detailed graphic like this Cellular Automata. A better solution would be to redraw the graphic in whatever space is available, which you can do with the `onResize` property:

var base = elasticSVG("#elastic_chart_A", {
    width: 475,
    aspect: 320 / 475,
    onResize: function(w, h, z, svg) {
        drawCA(svg, {
            ruleNumber: 90,
            squareSize: 7,
            margin: 2
        });
    }
});
